## üìù File Naming Convention

The documentation follows this naming pattern:

- `00-` prefix: Getting started
- `01-10`: Core topics in order
- `11-20`: Advanced topics
- `21-30`: Specialized topics
- `100+`: Comprehensive guides

## üîÑ Keeping Docs Updated

When making changes to the codebase:

1. **Configuration changes**: Update `09-configuration-reference.md`
2. **New API endpoints**: Update `14-api-routes-complete.md`
3. **New modules**: Add to `05-modules-detailed.md`
4. **Database changes**: Update `02-database-design.md`
5. **New dependencies**: Update `00-tech-stack-overview.md`
